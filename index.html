Scraping · Honest Reality

The Ingest URL button uses a public CORS proxy. Some sites work great; some block scrapers. Here's the honest picture so you know what to expect.

✅ Scraping Works Reliably

  • Grants.gov API (live federal grant search — use the Search tab, no scraping needed)
  • Foundation pages (Candid, FoundationCenter, Council on Foundations)
  • Corporate grant pages (FedEx Small Business, Comcast RISE, Verizon, Hello Alice)
  • State arts/economic dev (Texas Commission on the Arts, Texas Governor's Office)
  • USDA, DOE, HUD, NSF landing pages
  • LaunchSA, San Antonio Economic Development
  • Most simple .gov pages with the opportunity text on a single URL

⚠️ Hit or Miss (have manual fallback ready)

  • SBA grant pages (heavy JavaScript)
  • Cloudflare-protected pages (returns blocked HTML)
  • Sites with cookie walls or "I am human" checks
  • Long PDFs linked from a page (text not in the HTML)
  • Grants.gov opportunity detail pages (use the API in Search tab instead)

❌ Won't Work (and the fix)

  • Submittable, GrantHub, FluidReview — require login → fix = Chrome Extension (Path C, future)
  • Anything behind your personal account → fix = Chrome Extension
  • PDF-only opportunity announcements → fix = download PDF, drop into claude.ai, ask for structured extraction, paste back into Notes
  • State portals with captcha → fix = manual copy/paste into Notes, AI Draft still works

When Scraping Fails — The Manual Path Still Works

Even if Ingest URL fails on a site, you can: (1) open the grant page yourself, (2) copy the full description text, (3) click Edit on the grant in your pipeline, (4) paste the description into Notes, (5) click Vet Fit and AI Draft — both work using just the Notes field. Scraping is convenience, not a hard requirement.
